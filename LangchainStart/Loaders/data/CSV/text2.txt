### However, there is still a weakness

Your Gmail tool currently returns:

```python
{
    "message_id": ...,
    "subject": ...,
    "from": ...,
    "date": ...,
    "snippet": ...
}
```

A Gmail snippet is only a small preview.

For example:

```text
Apply to jobs at Wipro, Infosys...
```

The actual email may contain:

* Salary
* Location
* Skills
* Deadline
* Application link

but the agent never sees that.

Create another MCP tool:

```python
@mcp.tool()
def get_email_content(message_id: str):
```

that returns:

```python
{
    "subject": subject,
    "from": sender,

```

Then the agent workflow becomes:

```text
User
 ↓
get_recent_emails()
 ↓
Agent sees email list
 ↓
If user wants details
 ↓
get_email_content(message_id)
 ↓
Summarize
```

This is how real email assistants work.
